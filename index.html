操作系统做了什么

操作系统执行一个C语言的程序会发生什么？？

  • 用户:告诉操作系统执行程序
    • 通过命令行
    • 点击图标
  • OS
    • 找到程序的相关信息
    • 检查其类型是否为可执行文件
    • 通过程序首部信息，确定代码和数据在可执行文件中的位置，计算出对应的磁盘块地址
  • OS
    • 创建一个新的进程，将可执行文件映射到该进程结构，表示由该进程执行程序
    • 为程序设置CPU上下文环境并跳到程序开始初。
  • 执行程序第一条指令，发生缺页异常
  • OS:分配一页物理内存，并将代码从磁盘读入内存然后执行程序
  • 执行puts函数，在显示器上写字符串
  • OS:找到待送往的设备，通常设备由一个进程控制。所以OS把字符串送给进程
  • 显示器上显示相应字符串…(省略硬件过程)

CPU 时而执行用户程序，时而执行操作系统程序。

操作系统的定义和作用

操作系统是什么？

操作系统是一个系统软件，一些程序模块的集合。

  • 尽量有效合理的组织和管理计算机软硬件资源
  • 合理组织工作流程、控制程序的执行、向用户提供各种服务功能。
  • 使用户能够灵活方便的使用计算机。使整个计算机系统高效运行。

KEY WORD

  • 有效: 系统效率，资源利用率
  • 合理: 公平合理管理资源
  • 方便使用: 用户界面；编程接口

操作系统的三个作用

  • 资源的提供者 -> 有效
  • 向用户提供各种服务 -> 方便使用
  • 对硬件机器的扩展 -> 扩展能力

OS是资源的管理者

自底向上的来说，操作系统负责管理资源。资源包括硬件资源和软件资源。

怎样管理资源？？

  • 跟踪记录资源的使用情况 – 数据结构
  • 确定资源分配策略 – 算法
    • 动态 & 静态
  • 实施资源的分配和回收
  • 提高资源利用率
  • 保护资源的使用
  • 协调多个进程对资源请求的冲突

从资源管理的角度 - 五大基本功能

  1. 进程/线程管理
  2. 存储管理
  3. 文件管理
  4. 设备管理
  5. 用户结构

OS是各种系统服务的提供者

从用户角度来看，操作系统为用户提供了一组强大的方便使用的命令或者系统调用

OS是硬件之上的第一层软件

相比应用程序来说，与硬件相关的工作复杂繁琐。所以需要操作系统帮助我们进行抽象
应用程序
虚拟机器界面
操作系统
物理机器界面
硬件

OS对硬件机器的扩展

操作系统在应用程序和硬件之间建立了一个等价的扩展机器
作用

  • 对硬件抽象，提高可移植性
  • 笔底层硬件更容易编程

总结

操作系统的三个作用

  • 有效地管理资源
  • 通过命令接口、编程接口等为用户提供各种功能服务
  • 对硬件机器的扩展
