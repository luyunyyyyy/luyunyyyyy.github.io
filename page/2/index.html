寻找子树-算法-1

发表于 2018-01-15
字数统计: 974 字 | 阅读时长 ≈ 3 分钟

算法第一题 – 在大的树中寻找子树

问题定义

有两个不同大小的二叉树： T1 有上百万的节点； T2 有好几百的节点。请设计一种算法，判定 T2 是否为 T1的子树。
值得注意的地方

  • T1有上百万节点 – 可能后期需要考虑优化，若使用递归会不会导致递归层数过多。
  • 什么叫子树（这是一个大坑）
  • 待续

思路

关于树的题一开始肯定是想到递归。这个题比较自然的想法就是使用递归的方式来查找。
判断当前的树是否相同，若相同则分别查看子树是否相同，若不相同，则分别再对子树进行搜索。

初期的错误

但是我在第一次写的时候犯了一个错误。直接写了一个函数，然后来调用自身。后来发现跑不通。回想了一下，好像通常递归的函数会有一个类似 driver的函数来调用递归的函数。
还有一个错误就是没有想清楚递归的方式和每次递归要做的事情。

##写代码时需要注意的事情
一定要注意空指针或者 NULL
在 Java 中需要注意 null 的因素。同时，不光在写程序的时候，注意调用对象方法的时候，该对象是不是 null。还要注意问题的边界情况，输入的数据会不会是 null ，如果是 null 会不会是某种特别情况。
在该题中，若 T2 是 null ，则说明 T2是一个空树。可以认为空树是任何树的一个子树。这时候就要注意这种情况。

过程中遇到的问题

最重要的就是对于子树的定义问题！！
我认为的子树：只要有一部分结构相同即可。例如T2是T1中间的一部分
题目认为的子树：必须完全相同，即1和2相同的节点的儿子均相同，即为空则都为空。

解题思路

//上图
该题可以分解为两个部分

  • 对树进行遍历：Traverse方法
  • 判断T1和T2两个树是否相同，相同即为子树。isSubTree
    那么可以分为两个函数：一个函数 Traverse 负责遍历树。即当T1的不为空的时候，对当前根节点调用isSubTree。若当前节点下的树同T2不相同，则分别对T1的左右子树调用isSubTree。
    isSubTree函数的主要职责是判断两个树是否相同。这里需要注意传入参数为空的情况。因为是在T1中寻找T2，所以存在在运行过程中，T2为null的情况。这里需要注意，在isSubTree函数中，当T1等于T2等于null的时候，是结束的情况，说明两个树完全相同，到达最底层节点。当T1和T2中，有一个为null时，说明不相同返回false。当节点的值不相同的时候说明肯定为假。最后返回一个递归的，同时对左右子树调用的表达式。//此处应重构
    值得注意的是需要分辨在两个函数递归调用的过程中什么时候是T1传入子树，T2不变。什么时候T2也要传入相同的子树。

后续的内容

以后争取每次刷题都要写一个总结的内容，尤其是中间遇到很大问题的。同时还要在leetcode上找一下相关的题。有很多大神的黑魔法。在写这个题的时候遇到了点问题然后发现大神真的厉害。直接把树变成字符串然后使用子串查找的方法。
